## Agent 功能概述

本程序是一个基于大语言模型（LLM）的自动化 Agent，能够与 `utils.py` 工具集配合，通过多轮对话执行文件操作任务。其核心逻辑如下：

- **连接与配对**：通过 WebSocket 连接指定的 LLM 服务（如 DeepSeek），并独占一个浏览器实例。
- **消息构建**：合并 `SYSTEM_PROMPT.txt` 中的系统提示与用户输入（命令行参数 / `MESSAGE.txt` / `MESSAGE_DEFAULT.txt`）作为初始对话内容。
- **主循环**：持续与 LLM 交互，直到检测到任务完成标记。
  - **暂停控制**：每次循环前检查 `.pause` 文件，存在则等待删除，实现人工干预。
  - **命令识别**：从 LLM 回复中提取第一个以 `py utils.py` 开头的行作为待执行命令。
  - **工具执行**：执行命令，捕获输出，并根据输出内容决定后续行为：
    - 若输出以 `PAUSED:` 开头，则清空 `MESSAGE.txt`，等待 `.pause` 文件删除后恢复（恢复时优先读取用户写入的新指令）。
    - 否则将输出写入 `MESSAGE.txt` 作为下一轮输入。
  - **中间内容保存**：若回复不含命令且不含结束标记，则将整个回复保存到 `LAST_RESPONSE.txt`，并写入默认提示到 `MESSAGE.txt`，引导 LLM 继续。
  - **任务完成**：若回复中包含 ``（建议单独成行），则保存最终答案并退出循环。
- **日志记录**：每轮交互的发送内容、推理、回复及工具执行输出均追加到 `agent.log`，便于追踪。

## 使用方法

### 1. 准备文件
- **`SYSTEM_PROMPT.txt`**（可选）：定义系统提示，如角色设定、任务分解步骤、结束标记约定等。内容将附加在每轮消息前（仅第一轮合并用户消息）。
- **`MESSAGE.txt`**：用于存放用户输入或工具输出的中间结果。程序启动时会读取并清空该文件。
- **`MESSAGE_DEFAULT.txt`**（可选）：若未提供命令行参数且 `MESSAGE.txt` 为空，则读取此文件作为默认用户消息。
- **`.pause`**（控制文件）：存在时 Agent 暂停循环，删除后恢复。可由工具 `pause` 创建，或用户手动操作。
- **`LAST_RESPONSE.txt`**：用于保存 LLM 输出的非命令内容（如待写入文件的内容），供 `write` 等工具使用。

### 2. 运行命令
```bash
python agent.py [可选消息]
```
- 若提供命令行参数，则优先作为用户消息。
- 否则读取 `MESSAGE.txt` 并清空。
- 若仍为空，则读取 `MESSAGE_DEFAULT.txt`。

### 3. 与 LLM 交互规范
- **输出命令**：每条回复中应包含一行以 `py utils.py` 开头的命令，程序将执行该命令（忽略其他行）。例如：
  ```
  py utils.py read config.json
  ```
- **输出中间内容**：如需生成文件内容（如为 `write` 准备数据），直接输出纯文本（不含命令），程序会将其保存到 `LAST_RESPONSE.txt`，并自动提示 LLM 继续。
- **结束任务**：任务完成时，在回复中单独一行输出 ``，程序将保存最终答案并退出。
- **暂停请求**：当需要人工干预时，输出命令 `py utils.py pause`，工具将创建 `.pause` 文件并返回以 `PAUSED:` 开头的输出，Agent 暂停等待。用户解决问题后执行 `py utils.py resume` 或手动删除 `.pause` 文件即可恢复。

### 4. 注意事项
- **命令必须独占一行**，以 `py utils.py` 开头，程序仅识别第一个匹配行。
- **结束标记建议单独成行**，避免被误认为普通内容。程序通过 `in` 判断，但单独成行更清晰。
- **路径安全**：所有文件操作均基于 `.env` 中定义的根目录，防止路径逃逸。
- **日志文件**：`agent.log` 会持续增长，可定期清理。

## 典型工作流示例

1. **初始化**：编写 `SYSTEM_PROMPT.txt`，描述任务目标（如“生成项目文件结构”）。运行 `python agent.py`。
2. **LLM 输出命令**：例如 `py utils.py list src`，Agent 执行并返回目录列表。
3. **生成文件内容**：LLM 输出多文件内容（纯文本），Agent 保存到 `LAST_RESPONSE.txt`。
4. **执行写入**：LLM 输出 `py utils.py write_multiple`，工具从 `LAST_RESPONSE.txt` 解析并创建多个文件。
5. **暂停干预**：遇到复杂错误，LLM 输出 `py utils.py pause`，Agent 暂停。用户手动修复后删除 `.pause`，Agent 恢复。
6. **结束**：任务完成，LLM 输出 ``，Agent 保存最终输出并退出。

此 Agent 实现了自动化任务与人工干预的平滑结合，适用于需要文件操作和 LLM 协作的场景。

通过在输出中包含
```
py utils.py read utils.md
```
来获得工具使用的帮助
