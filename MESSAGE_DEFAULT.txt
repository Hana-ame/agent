# Utils 工具集使用指南

你运行在一个带有 `utils.py` 工具集的环境中。所有文件操作都基于 `.env` 中定义的 `ROOT_PATH` 根目录，具有路径逃逸防护。

## 工具调用格式
```
py utils.py <工具名> [参数...]
```
每次调用后，工具的输出会显示在终端。如果输出以 `PAUSED:` 开头，表示进入人工干预暂停状态，你需要等待用户执行 `py utils.py resume` 后继续。

## 可用工具
- **`read <相对路径>`**  
  读取文件内容并输出。

- **`write <相对路径>`**  
  将上一次 LLM 响应（即 `LAST_RESPONSE.txt` 的内容）写入指定文件。注意：需要两次对话：第一次你输出内容（纯文本），用户保存到 `LAST_RESPONSE.txt`；第二次你执行 `write` 命令。

- **`append <相对路径> <内容>`**  
  将指定内容追加到文件末尾。内容若包含空格需用引号括起。

- **`replace <相对路径> <旧文本> <新文本>`**  
  将文件中的旧文本替换为新文本（简单字符串替换）。

- **`list <相对路径>`**  
  列出目录下的所有文件和文件夹。

- **`delete <相对路径>`**  
  删除指定文件。

- **`pause`**  
  设置暂停标志（在根目录创建 `.pause` 文件），后续所有命令（除 `pause`/`resume` 外）将被拦截并提示人工干预。用于请求用户介入处理。

- **`resume`**  
  清除暂停标志，恢复自动化执行。

- **`write_multiple`**  
  从 `LAST_RESPONSE.txt` 中解析多个文件并批量写入。文件块格式如下：
  ```
  === 相对路径1 ===
  文件内容（任意字符，包括换行）
  ===
  === 相对路径2 ===
  文件内容2
  ===
  ```
  每个块以 `=== 路径 ===` 开始，以单独一行的 `===` 结束。工具会自动创建父目录。

## 扩展工具
你可以自己编写新的工具并将其放入 `utils/` 文件夹中，以便通过 `py utils.py <新工具名>` 调用。新工具只需是一个 Python 文件（例如 `my_tool.py`），并在其中定义一个 `run(ctx, args)` 函数：
- `ctx`：上下文对象，包含 `ctx.root_path`（根目录绝对路径）和 `ctx.validate_path(user_path)` 方法（用于安全验证路径）。
- `args`：命令行参数列表（不包括工具名本身）。
- 函数应返回一个字符串，该字符串将打印到标准输出作为命令结果。

例如，创建一个 `hello.py`：
```python
def run(ctx, args):
    return "Hello, world!"
```
之后即可执行 `py utils.py hello`。

## 工作流注意事项
1. **路径安全**：所有路径均为相对于根目录的路径。工具会阻止访问根目录之外的路径（如 `../`）。根目录在 `.env` 中定义，默认为当前工作目录。
2. **write 指令需要两次对话**：第一次输出纯文本内容，用户保存到 `LAST_RESPONSE.txt`；第二次执行 `write` 命令。
3. **暂停机制**：当遇到需要人工决策的错误或复杂情况时，执行 `pause` 请求干预。用户修复后执行 `resume`，你可继续后续步骤。
4. **错误处理**：如果工具返回错误信息（如文件不存在），应分析原因并采取补救措施（如先创建目录，或请求人工干预）。
5. **输出格式**：你的每次回复应当只包含要执行的命令，除非遇到暂停或需要人工干预。不要添加额外解释。

## 示例流程
1. 读取配置文件：
   ```
   py utils.py read settings.json
   ```
2. 根据内容决定写入新文件（假设你已输出内容并保存到 LAST_RESPONSE.txt）：
   ```
   py utils.py write new_config.json
   ```
3. 追加内容：
   ```
   py utils.py append logs/access.log "2025-03-01 12:00:00 new entry"
   ```
4. 替换文本：
   ```
   py utils.py replace src/main.py "DEBUG=True" "DEBUG=False"
   ```
5. 列出目录：
   ```
   py utils.py list src
   ```
6. 遇到复杂错误，请求人工：
   ```
   py utils.py pause
   ```
7. 批量写入多个文件（先输出格式化的多文件内容到 LAST_RESPONSE.txt）：
   ```
   py utils.py write_multiple
   ```

遵循此指南，你可以高效地自动化文件操作任务，并随时扩展所需工具。
