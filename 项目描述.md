# 项目描述

这是一个多轮对话的 Agent 项目，主文件为 `agent.py`（或 `agent_v2.py`）。项目通过 `utils/` 目录下的工具与系统交互，所有工具均通过 `py utils.py <工具名> [参数...]` 调用。

## 核心原则

- **先观察，再设计，后修改，以验证收尾**：在修改任何文件前，先用 `ls`、`cat` 等工具了解当前状态；修改后用 `git status`、测试命令等验证。
- **版本控制**：每次完成有意义的修改后，立即使用 `git add` 和 `git commit` 提交，确保历史可追溯。
- **不要修改自动生成的文件**：`.agent/` 目录下的文件（如 `THIS_RESPONSE.txt`、`LAST_RESPONSE.txt`、`THOUGHTS.md` 等）由系统自动管理，请勿直接编辑。
- **命令格式**：所有命令中的参数之间用空格分隔，不要有多余空格缩进。

## 可用工具

执行 `py utils.py help` 可查看所有工具列表，使用 `py utils.py help <工具名>` 查看具体用法。

### 常用工具

- **`cat`**：查看一个或多个文件内容（支持统一格式输出，自动添加文件标记）。
- **`write`**：覆盖写入文件（自动创建父目录，成功时返回内容预览）。
- **`write_multiple`**：从响应文件（`.agent/THIS_RESPONSE.txt`）批量写入多个文件，格式为：
  ```
  === 相对路径1 ===
  内容
  === end of 相对路径1 ===
  === 相对路径2 ===
  内容
  === end of 相对路径2 ===
  ```
- **`ls`**：列出目录内容，目录后加 `/`。
- **`delete`**：删除一个或多个文件。
- **`cp`**：复制文件或目录（支持 `-r` 递归）。
- **`mv`**：移动/重命名文件或目录。
- **`replace`**：替换文件中的文本（简单字符串替换）。
- **`git`**：执行 Git 命令，输出格式统一（成功返回命令输出，失败返回错误块）。
- **`thoughts`**：管理 `.agent/THOUGHTS.md` 中的想法：
  - `pop`：弹出第一条想法（并删除）
  - `peek`：查看第一条想法（不删除）
  - `list`：列出所有想法（带编号）
  - `add <想法>`：添加一条新想法
  - `clear`：清空所有想法
- **`snippet`**：管理文件中的代码段（格式 `# [START] name` / `# [END] name`，支持多语言注释符）：
  - `list <file>`：列出文件中的所有代码段
  - `get <file> <name>`：获取指定代码段的内容
  - `set <file> <name> --content <内容>` 或 `--file <文件>`：设置代码段（创建或更新）
  - `delete <file> <name>`：删除指定代码段
- **`apply_snippet_block`**：从响应文件中读取代码段块，自动更新到对应文件的代码段中（格式与 `write_multiple` 类似，但块内包含代码段标记）。
- **`pause`**：在根目录创建 `.pause` 文件，暂停 Agent 循环。
- **`resume`**：删除 `.pause` 文件，恢复 Agent 执行。
- **`time`**：显示当前时间或管理计时器（`--date`、`--time`、`--timestamp`、`--start`、`--stop`、`--elapsed`、`--format`）。

## 工作流程示例

1. 查看当前目录结构：
   ```
   py utils.py ls
   ```

2. 查看某个文件内容：
   ```
   py utils.py cat path/to/file
   ```

3. 设计修改方案，然后写入文件：
   - 使用 `write` 写入单个文件（多行内容可直接输入，以 Ctrl+Z 或 Ctrl+D 结束）：
     ```
     py utils.py write new_file.txt
     [粘贴内容，然后 Ctrl+Z]
     ```
   - 或使用 `write_multiple` 批量写入：
     先在 `.agent/THIS_RESPONSE.txt` 中准备格式（可借助 `write` 写入该文件）：
     ```
     py utils.py write .agent/THIS_RESPONSE.txt
     === file1.txt ===
     content1
     === end of file1.txt ===
     === file2.txt ===
     content2
     === end of file2.txt ===
     ```
     然后执行：
     ```
     py utils.py write_multiple
     ```

4. 验证修改：
   ```
   py utils.py cat 修改的文件
   ```
   若涉及命令执行，直接运行命令检查结果。

5. 提交修改：
   ```
   py utils.py git add 修改的文件
   py utils.py git commit -m "描述修改"
   py utils.py git status   # 确认工作区干净
   ```

6. 若无后续任务，调用 `py utils.py pause`；若有想法继续，调用 `py utils.py thoughts pop` 获取下一个任务。

## 代码段操作详细说明

### 定义代码段
在文件中使用以下格式定义代码段（注释符根据文件扩展名自动识别，如 `.py` 用 `#`，`.js`/`.go` 用 `//`，`.sql` 用 `--` 等）：
```
# [START] 代码段名称
代码内容
# [END] 代码段名称
```
或
```
// [START] 代码段名称
代码内容
// [END] 代码段名称
```

### 通过响应文件批量更新代码段
在 `.agent/THIS_RESPONSE.txt` 中按以下格式组织：
```
=== 文件路径 ===
# [START] 代码段名称
新内容
# [END] 代码段名称
=== end of 文件路径 ===
```
然后执行：
```
py utils.py apply_snippet_block
```
工具会自动根据文件路径判断注释符，并更新相应文件的代码段（若文件不存在则创建）。

## 注意事项

- **命令格式**：所有命令中的参数之间用空格分隔，不要有多余空格缩进。
- **多行内容**：使用 `write` 写入多行内容时，可直接输入多行文本，以 Ctrl+Z (Windows) 或 Ctrl+D (Linux/macOS) 结束。
- **响应文件**：`THIS_RESPONSE.txt` 和 `LAST_RESPONSE.txt` 由系统管理，用于传递批量数据，请勿手动修改（除非明确需要）。
- **想法管理**：通过 `thoughts add` 添加新任务，`thoughts pop` 获取下一个任务，保持工作流连续。
- **版本控制**：每次完成修改后务必提交，确保工作区干净，否则 `thoughts pop` 可能拒绝执行。
