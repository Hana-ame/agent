# 项目报告：基于LLM的自动化Agent与文件操作工具集

## 1. 项目概述

本项目是一个基于大语言模型（LLM）的自动化智能体（Agent），它通过与一个自定义工具集 `utils.py` 的紧密协作，实现多轮对话驱动的文件操作任务自动化。核心设计思路是：将LLM作为“决策大脑”，工具集作为“执行手脚”，通过WebSocket连接LLM服务（如DeepSeek），在循环中不断解析LLM的回复、执行相应命令、并将结果反馈给LLM，直至任务完成。

项目特别强调**人机协同**：当遇到需要人工判断的复杂情况时，Agent能够主动暂停，等待用户干预后继续；同时，用户也可随时介入修改指令或文件，体现了良好的灵活性和可控性。

## 2. 核心功能

### 2.1 智能对话与任务分解
- 通过WebSocket与LLM服务建立独占连接，支持将系统提示（`SYSTEM_PROMPT.txt`）与用户消息合并，引导LLM逐步分解任务并生成可执行命令。
- 系统提示中可约定任务结束标记（如 `=== FINISH ===`），当LLM输出该标记时，Agent自动终止循环并保存最终答案。

### 2.2 命令识别与执行
- 从LLM回复中提取第一个以 `py utils.py` 开头的行作为待执行命令，其余内容忽略。
- 执行命令后捕获输出，并根据输出内容决定后续流程：
  - 若输出以 `PAUSED:` 开头，则进入暂停状态，等待人工干预。
  - 否则将输出写入 `MESSAGE.txt`，作为下一轮对话的输入。
- 若回复中不含命令且无结束标记，则将整个回复保存到 `LAST_RESPONSE.txt`，并自动写入默认提示到 `MESSAGE.txt`，引导LLM继续。

### 2.3 暂停与恢复机制
- 通过文件 `.pause` 控制Agent的循环：存在时暂停，删除后恢复。
- 工具 `pause` 可在根目录创建 `.pause` 文件并返回以 `PAUSED:` 开头的输出，触发Agent等待。
- 工具 `resume` 删除 `.pause` 文件，使Agent恢复运行。
- 此机制为人工介入复杂错误、修改指令或检查中间结果提供了标准接口。

### 2.4 丰富的文件操作工具集
`utils.py` 提供了一系列安全的文件操作工具，所有路径均基于 `.env` 中定义的 `ROOT_PATH`，防止路径逃逸：

- **文件读写**：`read`、`write`、`append`、`replace`
- **目录操作**：`list`、`delete`
- **批量写入**：`write_multiple` 支持从 `LAST_RESPONSE.txt` 解析多文件块并批量创建
- **暂停控制**：`pause` / `resume`
- **可扩展性**：支持用户自定义工具放入 `utils/` 文件夹，只需实现 `run(ctx, args)` 接口即可无缝集成。

### 2.5 完整的日志追踪
- 每轮交互（发送内容、LLM推理、回复、工具执行输出）均追加到 `agent.log`，便于调试、复盘和审计。

## 3. 使用方法

### 3.1 准备工作
- **环境配置**：创建 `.env` 文件，定义 `ROOT_PATH` 作为所有文件操作的根目录（默认为当前工作目录）。
- **系统提示**：可选编写 `SYSTEM_PROMPT.txt`，设定角色、任务分解规则、结束标记等。
- **用户消息**：可通过命令行参数、`MESSAGE.txt` 或 `MESSAGE_DEFAULT.txt` 提供初始任务描述。
- **控制文件**：`.pause` 用于手动控制暂停（可选）。

### 3.2 运行Agent
```bash
python agent.py [可选消息]
```
- 优先使用命令行参数作为用户消息；
- 若无参数，则读取 `MESSAGE.txt` 并清空；
- 若 `MESSAGE.txt` 为空，则读取 `MESSAGE_DEFAULT.txt`。

### 3.3 与LLM交互规范
- **输出命令**：每条回复中必须包含一行以 `py utils.py` 开头的命令，Agent将执行该命令（仅第一行有效）。
- **输出内容**：如需生成文件内容（如为 `write` 准备数据），直接输出纯文本，Agent会将其保存到 `LAST_RESPONSE.txt`，并自动提示LLM继续。
- **结束任务**：任务完成时，在回复中单独一行输出 `=== FINISH ===`，Agent保存最终答案并退出。
- **暂停请求**：当需要人工干预时，执行命令 `py utils.py pause`，工具创建 `.pause` 文件并返回 `PAUSED:` 输出，Agent暂停等待。用户解决问题后执行 `py utils.py resume` 或手动删除 `.pause` 即可恢复。

### 3.4 典型工作流示例
1. **初始化任务**：编写 `SYSTEM_PROMPT.txt`（例如：“生成一个React项目的文件结构”），运行 `python agent.py`。
2. **LLM输出命令**：`py utils.py list src` → Agent执行并返回目录列表。
3. **生成文件内容**：LLM输出多文件内容（纯文本）→ Agent保存到 `LAST_RESPONSE.txt`。
4. **执行写入**：LLM输出 `py utils.py write_multiple` → 工具从 `LAST_RESPONSE.txt` 解析并创建多个文件。
5. **暂停干预**：遇到复杂错误，LLM输出 `py utils.py pause` → Agent暂停，用户手动修复后删除 `.pause` 继续。
6. **结束**：任务完成，LLM输出 `=== FINISH ===` → Agent保存最终输出并退出。

## 4. 改进策略

尽管现有设计已具备较强的自动化和灵活性，但仍可从以下几个方面优化：

### 4.1 命令解析与执行增强
- **多命令支持**：当前仅识别第一行命令，可扩展为支持多条命令顺序执行，提高效率。
- **命令参数校验**：增加对命令格式和参数的校验，提前发现错误并引导LLM修正。
- **异步执行**：对于耗时工具（如批量写入），可考虑异步执行以避免阻塞对话循环。

### 4.2 错误处理与恢复机制
- **细粒度错误分类**：工具返回错误时，可附加错误类型（如文件不存在、权限不足），帮助LLM更精准地决策。
- **自动重试策略**：对于临时性错误（如网络波动），允许配置自动重试次数。
- **状态持久化**：在暂停时保存当前对话状态，恢复时能够无缝继续，避免重新推理。

### 4.3 安全性加固
- **路径白名单**：除根目录限制外，可增加对特定敏感目录（如 `.git`、`node_modules`）的访问控制。
- **命令注入防护**：对LLM输出的命令进行严格过滤，确保仅允许预定义的工具名，防止恶意构造。
- **速率限制**：对工具调用频率进行限制，避免因LLM错误循环导致资源耗尽。

### 4.4 用户体验优化
- **Web界面**：提供简单的Web控制台，实时显示对话日志、暂停状态，并允许手动发送指令。
- **可视化暂停**：暂停时可通过系统通知或Web提示醒目地告知用户，并提供一键恢复按钮。
- **更友好的日志**：支持日志分级（DEBUG/INFO/ERROR），并可配置日志轮转，避免单个文件过大。

### 4.5 扩展性与集成
- **工具自动发现**：`utils/` 目录下的工具可自动注册，无需硬编码到主程序。
- **插件系统**：允许第三方开发者贡献工具包，并通过配置文件动态加载。
- **与其他系统集成**：支持将最终结果推送到外部API、发送邮件或存储到数据库。

### 4.6 性能与资源优化
- **连接复用**：当前WebSocket独占一个浏览器实例，可考虑连接池以支持多任务并发。
- **缓存机制**：对频繁读取的文件内容（如配置文件）进行缓存，减少磁盘I/O。
- **流式输出**：LLM推理过程中支持流式输出，提升交互实时性。

通过以上改进，该项目可进一步演变为一个更健壮、易用且可扩展的自动化工作流引擎，适用于开发辅助、文档生成、批量处理等多种场景。
