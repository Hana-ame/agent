# 工具测试报告：append.py 和 write.py

## 测试时间
2026-03-12

## append.py

### 期望用法
命令格式：`py utils.py append <相对路径> <内容>`
- 如果文件不存在，自动创建。
- 自动处理换行：如果文件非空且不以换行结尾，则先添加换行再追加。
- 成功输出：`{rel_path}中被追加了以下内容\n{preview}`
- 失败输出统一格式错误块：
  ```
  === 路径 ===
  错误：具体错误信息
  === end of 路径 ===
  ```

### 实际用法（修改后）
与期望一致。

### 测试结果

#### 1. 追加到已有文件
命令：
```
py utils.py write test_append.txt "First line"
py utils.py append test_append.txt "Second line"
py utils.py cat test_append.txt
```
输出：
```
test_append.txt中被写入了以下内容
First line
test_append.txt中被追加了以下内容
Second line
=== test_append.txt ===
First line
Second line
=== end of test_append.txt ===
```
✅ 正确追加，内容符合预期。

#### 2. 追加到新文件
命令：`py utils.py append new_file.txt "Content"`
输出：`new_file.txt中被追加了以下内容\nContent`
✅ 文件自动创建，内容写入。

#### 3. 追加到深层路径（自动创建父目录）
命令：`py utils.py append non/existent/file.txt "Test"`
输出：`non/existent/file.txt中被追加了以下内容\nTest`
✅ 父目录自动创建，文件写入成功。
