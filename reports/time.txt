# 工具测试报告：time.py

## 测试时间
2026-03-12

## 期望用法
命令格式：`py utils.py time [选项]`

选项：
  --date                只显示当前日期
  --time                只显示当前时间
  --timestamp           显示 Unix 时间戳
  --start               开始计时
  --stop                停止计时并显示经过时间
  --elapsed             显示当前经过时间（不停止）
  --format <格式>       使用自定义 strftime 格式

输出格式：
  - 成功时：返回相应的文本。
  - 失败时：
    === time <子命令> ===
    错误：具体错误信息
    === end of time <子命令> ===

## 实际用法（修改后）
与期望一致。

## 测试结果

### 1. 显示当前日期时间
命令：`py utils.py time` → 输出 `2026-03-12 13:01:17` ✅

### 2. 显示当前日期
命令：`py utils.py time --date` → 输出 `2026-03-12` ✅

### 3. 显示当前时间
命令：`py utils.py time --time` → 输出 `13:01:18` ✅

### 4. 显示 Unix 时间戳
命令：`py utils.py time --timestamp` → 输出 `1773291678` ✅

### 5. 计时器启动
命令：`py utils.py time --start` → 输出 `计时器已启动。` ✅

### 6. 显示已过去时间
命令：`py utils.py time --elapsed` → 输出 `当前经过时间: 0小时 0分钟 0.11秒` ✅

### 7. 停止计时
命令：`py utils.py time --stop` → 输出 `经过时间: 0小时 0分钟 0.24秒` ✅

### 8. 停止后再停止（错误情况）
命令：`py utils.py time --stop` → 输出错误信息（因为计时器已停止）：
```
=== time stop ===
错误：没有正在运行的计时器。
=== end of time stop ===
```
✅ 错误块格式正确。

### 9. 自定义格式
命令：`py utils.py time --format "%Y年%m月%d日 %H时%M分%S秒"` → 输出 `2026年03月12日 13时01分18秒` ✅

### 10. 未知选项
命令：`py utils.py time --unknown` → 返回帮助信息并提示参数错误，错误块包含用法说明 ✅

### 11. 多个选项冲突
命令：`py utils.py time --format --date` → 返回错误信息：
```
=== time  ===
错误：参数错误，请检查用法。
...
```
✅ 正确捕获错误。

## 结论
`time.py` 已按统一风格修改并通过测试。
