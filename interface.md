# Agent 系统接口规范

## 1. LLM 客户端适配器接口 (BaseAdapter)
管理与不同大语言模型（如 WebSocket 专有协议或 OpenAI 兼容 HTTP API）的底层通信。

*   **`new_chat()`**
    *   **触发条件**：新建会话或清空历史记忆时调用。
    *   **输入**：无
    *   **输出**：无
*   **`new_prompt(prompt: str) -> Tuple[str, str]`**
    *   **触发条件**：发送新一轮用户输入以获取模型响应。
    *   **输入**：`prompt` (字符串，用户输入内容)
    *   **输出**：返回元组 `(reasoning, content)`，分别为模型的思考过程与最终输出内容。

## 2. 文本解析与执行规则接口 (BaseRule)
用于对模型返回的文本进行模式匹配并提取/执行动作（支持高度模块化扩展）。

*   **`match_and_execute(text: str) -> str`**
    *   **触发条件**：每次接收到完整的模型回复 `content` 后，由规则处理器逐一调用验证。
    *   **输入**：`text` (字符串，模型输出的完整文本)
    *   **输出**：返回提取与执行后的日志/结果汇总。若未匹配到任何内容，返回空字符串。

## 3. 工具箱加载与执行接口 (Utils Module)
封装独立于 Agent 的工具生态，支持在多路径下动态加载工具库。

*   **`get_tool_info(tool_name: str) -> dict`**
    *   **触发条件**：查询工具列表或特定工具帮助时调用。
    *   **输入**：`tool_name` (字符串，工具名或别名)
    *   **输出**：返回包含 `name`, `alias`, `description`, `doc` 的字典。
*   **`run(ctx: object, args: list) -> str` (定义于每个独立工具内)**
    *   **触发条件**：命令行或进程被拉起并命中对应工具时调用。
    *   **输入**：`ctx` (上下文对象，包含运行环境路径等), `args` (字符串列表，命令行参数)
    *   **输出**：返回工具执行结果字符串，发生异常需向上抛出。
