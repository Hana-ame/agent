### 主要修改说明
1. **匹配规则**：正则表达式 `\[([^\]]+)\]` 匹配所有方括号内的内容（相对路径），不限于 URL。
2. **基础目录获取**：
   - 优先使用命令行 `--base-dir`。
   - 其次尝试读取当前目录下的 `.env` 文件中的 `BASE_DIR` 变量（支持 `KEY=value` 格式，自动去除引号）。
   - 再尝试读取 `config.txt` 的第一行作为基础路径。
   - 最后默认使用当前工作目录。
3. **路径拼接与安全**：
   - 使用 `os.path.join` 和 `os.path.abspath` 获得绝对路径。
   - 增加安全检查，防止相对路径包含 `..` 跳出基础目录（可选，可根据需要注释掉）。
4. **文件读取**：以 UTF-8 编码读取，错误时返回错误信息。
5. **输出格式**：与原脚本一致，将内容包裹在 Markdown 代码块中。

### 使用示例
假设你的 Markdown 文件 `doc.md` 中有如下内容：
```markdown
这是本地文件内容：
[code/example.py]
```
你有一个基础目录 `/home/user/projects`，且该目录下存在 `code/example.py`。

运行命令：
```bash
python expand.py doc.md doc_expanded.md --base-dir /home/user/projects
```
生成的 `doc_expanded.md` 会包含：
```markdown
这是本地文件内容：
[code/example.py]

```
（这里插入 example.py 的内容）
```

如果你没有提供 `--base-dir`，脚本会依次检查 `.env` 和 `config.txt`，最后使用当前目录。

### 注意事项
- 请确保输入文件和输出文件不要相同，以免覆盖原始数据。
- 如果相对路径指向的文件不存在或无法读取，会在原位置插入错误提示。
- 默认启用了路径安全检查，如果相对路径试图访问基础目录之外的文件，操作会被拒绝并显示警告。如需禁用，可以注释掉 `if not full_path.startswith(...)` 那段代码。
